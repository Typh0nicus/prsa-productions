# Gregor Productions Portfolio Foundation

This site is a portfolio for Gregor Prsa's editing work. It should feel like a sharp one-person studio, not a generic creator agency.

## North Star

Translate Gregor's editing taste into a web experience: dark, quiet, cinematic, specific, and built around real work.

## Palette

Source: Gregor's profile photo: black hoodie, charcoal room, cyan monitor rim, faint violet spill.

- Base: `#03060b`
- Secondary base: `#07101a`
- Panel: `#0d1622`
- Raised panel: `#111b2a`
- Text: `#eef7ff`
- Muted text: `#9aaabd`
- Faint text: `#566477`
- Cyan accent: `#27d9f2`
- Screen blue: `#4f8dff`
- Violet trace: `#6f72ff`

Use cyan sparingly for state, line, and directional cues. Violet should be a trace, never a dominant theme.

## Content Rules

- Do not invent metrics, view counts, testimonials, client quotes, or role claims.
- Prefer real video embeds, real thumbnails, real project names, and explicit placeholders where data is missing.
- Avoid generic agency language: growth, premium, world-class, retention-first, algorithm, high-performing, full-service.
- Gregor Productions is a discreet one-person studio. The site can be confident, but it should not sound inflated.
- Replace any calendar or booking language unless a real booking URL exists.

## Design Rules

- The work should carry the page. UI should frame it, not compete with it.
- No glow soup, decorative blobs, fake scanlines, animated marquees, or template luxury tropes.
- Components should feel edited: strong crops, deliberate spacing, clear metadata, and restrained motion.
- Motion should feel like cuts, reveals, and timing, not generic web animation.
- Case studies are proof surfaces, not marketing pages.

## Phase 2 Visual Language

The site should read like a quiet edit-room index, not a creator-agency landing page.

- Voice: short, factual, spare. Prefer nouns and labels over sales sentences.
- Section labels: use archival/editorial terms: `Credits`, `Selected uploads`, `Notes`, `About`, `Contact`.
- Buttons: direct actions only: `View work`, `Email`, `Open project`, `Watch`.
- Metadata: project number, creator, format, role, source link. Avoid invented performance language.
- Surfaces: media is the hero. UI uses hairlines, black-blue panels, and small cyan state cues.
- Shapes: square or 6px radius for interface surfaces. Larger radius only when inherited from video frames.
- Motion: crisp reveal and hover timing. No decorative looping motion until the static layout is strong.
- Content must stay visible by default. Do not hide sections behind generic scroll-reveal effects.
- Cyan budget: use for current state, small marks, focus and interaction. Never as a page-wide wash.
